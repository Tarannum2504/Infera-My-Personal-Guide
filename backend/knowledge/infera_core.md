  Answer structure:
    SELECT MAX(salary) FROM employees
    WHERE salary < (SELECT MAX(salary) FROM employees);
  OR using DENSE_RANK:
    SELECT salary FROM (
      SELECT salary, DENSE_RANK() OVER
      (ORDER BY salary DESC) AS rnk FROM employees
    ) t WHERE rnk = 2;

  Answer structure:
    SELECT customer_id FROM orders
    WHERE YEAR(order_date) = 2024 AND MONTH(order_date) = 1
    AND customer_id NOT IN (
      SELECT customer_id FROM orders
      WHERE YEAR(order_date) = 2025 AND MONTH(order_date) = 1
    );

  Q3. Difference between RANK(), DENSE_RANK(), ROW_NUMBER()?
  Answer:
    ROW_NUMBER(): Unique sequential number. No ties. 1,2,3,4,5
    RANK()      : Ties get same rank. Gaps after ties. 1,1,3,4,5
    DENSE_RANK(): Ties get same rank. NO gaps. 1,1,2,3,4

  Q4. Find duplicate email addresses in a users table.
  Answer:
    SELECT email, COUNT(*) as count FROM users
    GROUP BY email HAVING COUNT(*) > 1;

  Q5. Write a self-join to find employees who earn more
      than their manager.
  Answer:
    SELECT e.name, e.salary FROM employees e
    JOIN employees m ON e.manager_id = m.emp_id
    WHERE e.salary > m.salary;

  Q6. What is a CTE? Write one to find top 3 products
      by sales.
  Answer:
    WITH ranked_products AS (
      SELECT product_id, SUM(sales) as total_sales,
      RANK() OVER (ORDER BY SUM(sales) DESC) as rnk
      FROM sales GROUP BY product_id
    )
    SELECT * FROM ranked_products WHERE rnk <= 3;

  Q7. Explain window functions with a business example.
  Answer script:
    "Window functions perform calculations across a set of
    rows related to the current row, without collapsing
    the result like GROUP BY does.
    Example: If I want to see each employee's salary AND
    the average salary of their department in the same row:
    SELECT name, salary, department,
    AVG(salary) OVER (PARTITION BY department) AS dept_avg
    FROM employees;
    This is useful in business for comparing individual
    performance against team or regional averages."

  Answer:
    def reverse_string(s):

        return result

  Q2. Find all duplicates in a list.
  Answer:
    def find_duplicates(lst):
        seen = set()
        duplicates = set()
        for item in lst:
            if item in seen:
                duplicates.add(item)
            seen.add(item)
        return list(duplicates)

  Q3. What is a generator? Write one that yields squares.
  Answer:
    "A generator is a function that returns an iterator.
    It uses 'yield' instead of 'return', which means it
    generates values one at a time, saving memory for large
    datasets — critical in data engineering pipelines."
    def square_generator(n):
        for i in range(n):
            yield i ** 2

  Q4. What is the difference between deepcopy and shallowcopy?
  Answer:
    "Shallow copy creates a new object but references the
    same nested objects. Deep copy creates completely
    independent copies of the object AND all nested objects.
    In data work: if I copy a Pandas DataFrame with shallow
    copy and modify it, the original may change. Deep copy
    is safer for data transformation pipelines."

  Q5. How do you handle missing values in Pandas?
  Answer:
    "Multiple approaches depending on business context:
    df.dropna() — Remove rows with missing values (use when
    missing data is random and small percentage)
    df.fillna(value) — Fill with constant (use for categorical)
    df.fillna(df.mean()) — Fill with mean (use for numerical)
    df.interpolate() — Interpolate (use for time series)
    I always first check: df.isnull().sum() to understand
    the scale of missing data before deciding."

TCS TECHNICAL INTERVIEW — ALL LIKELY QUESTIONS + ANSWERS

SQL Block:
  Q: Difference between WHERE and HAVING?
  A: "WHERE filters rows BEFORE aggregation.
     HAVING filters groups AFTER aggregation.
     Example: WHERE salary > 50000 filters individual rows.
     HAVING COUNT(*) > 5 filters department groups.
     You cannot use aggregate functions in WHERE clause."

  Q: What are the different types of JOINs?
  A: "INNER JOIN: Only matching rows from both tables.
     LEFT JOIN: All rows from left table + matching from right.
                Non-matching right rows show NULL.
     RIGHT JOIN: All rows from right + matching from left.
     FULL OUTER JOIN: All rows from both. NULLs where no match.
     CROSS JOIN: Cartesian product of both tables (all combinations).
     SELF JOIN: Table joined with itself — used for hierarchical data
                like finding employees and their managers."

  Q: What is indexing? When would you use it?
  A: "An index is a data structure that speeds up data retrieval
     by allowing the database to find rows without scanning every
     row in the table — like an index in a book vs reading cover to cover.
     I would use an index on:
     → Columns used frequently in WHERE clauses
     → Columns used in JOIN conditions
     → Foreign keys
     I would NOT index columns that are updated frequently, as
     indexes slow down INSERT/UPDATE/DELETE operations."

DBMS Block:
  Q: What is normalization? Explain 1NF, 2NF, 3NF.
  A: "Normalization is the process of organizing database tables
     to reduce redundancy and improve data integrity.
     1NF: Each column contains atomic (indivisible) values.
          No repeating groups. Each row is unique.
     2NF: Must be in 1NF + every non-key column must depend
          on the ENTIRE primary key (no partial dependencies).
          Relevant when there's a composite primary key.
     3NF: Must be in 2NF + no transitive dependencies.
          Non-key columns must depend ONLY on the primary key,
          not on other non-key columns.
     In practice: I design tables to at least 3NF for OLTP
     systems, but denormalize intentionally for OLAP/reporting."

  Q: What are ACID properties?
  A: "ACID ensures reliable database transactions:
     Atomicity: Transaction is all-or-nothing. Either all operations
                succeed or none do. If a bank transfer fails midway,
                money doesn't disappear.
     Consistency: Database moves from one valid state to another.
                  Rules and constraints are never violated.
     Isolation: Concurrent transactions don't interfere with each other.
                Like two people editing different rows simultaneously.
     Durability: Once committed, data persists even after system crash.
                 Achieved through write-ahead logging (WAL)."

OOP Block:
  Q: What is polymorphism? Give a Python example.
  A: "Polymorphism means 'many forms' — the same method name
     behaves differently based on the object calling it.
     Example:
     class Shape:
         def area(self): pass
     class Circle(Shape):
         def area(self): return 3.14 * r * r
     class Rectangle(Shape):
         def area(self): return l * w
     shapes = [Circle(5), Rectangle(4,6)]
     for s in shapes: print(s.area())
     The same 'area()' call produces different results based on
     the actual object type. In data analytics, polymorphism is
     useful when building data loaders for different file formats —
     CSV loader, JSON loader, Parquet loader all share the same
     interface but different implementations."

  Q: Difference between abstraction and encapsulation?
  A: "Encapsulation: Bundling data and methods that operate on
     that data within a single unit (class), and restricting
     direct access to internal data using private attributes.
     It's about HIDING implementation details.
     Abstraction: Hiding complex implementation and showing only
     the necessary interface to the user.
     Simple analogy: A car's engine is encapsulated under the hood.
     The steering wheel and accelerator are the abstraction — you
     use them without knowing how the engine works."
